# コンテンツマーケティング戦略: 2026年5月以降（v2.1）

対象: beekle.jp（プロジェクト管理 / システム開発支援を扱うB2Bコーポレートサイト）
情報源: GA4 (2026-01-01〜04-28) + **Search Console 実データ（過去90日）** + **DataForSEO 月間SV / CPC**（2026-04-29取得）+ 既存コラム

> **v2.1 改訂サマリ（2026-04-29 後段）**: v2 は SC の imp/click のみで優先度を組んでいたが、DataForSEO で月間SVを取得した結果、**SCの imp は部分一致拡張で過大に出ているケースが多く、実SVベースで再評価すると以下の構造的なズレ** が判明した。
>
> 1. **要件定義クラスター（v2でP3）が実SV最大（「要件定義」12,100/月）** → P1 に格上げ
> 2. **「要件定義 失敗」(SV30) より「DX 失敗」(SV260) / 「システム開発 失敗」(SV140) の方が3-5倍大きい** → 失敗予防ピラーの主軸を再設計
> 3. **AI クラスターは「生成AI 受託開発」(SV30) 単独では弱い** → 「AI受託開発」(260) ＋「生成AI 開発」(390) ＋「AI システム開発」(480) を一括カバーする再構成
> 4. **CDP のロングテール（コンポーザブルCDP, CDP データパイプライン等）は SV取得不能** → ピラー記事内のセクションに統合し、独立記事は廃止
> 5. **「PMO とは」(18,100) / 「DX 推進」(4,400) は SV巨大だがインテントが教育需要中心 → CV弱と判断し据置**

> **v2 改訂サマリ（2026-04-29）**: 初版はSC未連携前提で「自社実績ページの強さ＝ニーズの実証」として推測ベースの戦略を組んでいた。今回SC実データを取得した結果、**初版の優先度設定が実需と大きくズレていた**ため全面改訂。

### 2.1 P0: インデックス改善（最優先・全施策の前提）

1. **5xx エラー22ページの調査・修正**（SSRレンダリング失敗の可能性高）

### 2.8 見積もり/費用クラスター（SEO優先度↓ / CV補助として維持）

v1で「P2最優先」としていたが、検索流入は10impしかない。一方、SVベースで見ると **「システム開発 見積もり」260/月で CPC $27.27** = **全KWで最高単価**。少SVでも CV直結＆社内稟議補助として価値が高い。
**B2B検討プロセスの最終段階（発注前の比較・社内稟議）で読まれる「補助コンテンツ」としての価値は別軸**。SEO起点では伸ばさないが、以下の役割で維持する:

**役割の再定義**:
- 商談化フェーズの安心材料（指名検索 or 営業から共有されたURL経由で読まれる）
- 他クラスター記事からの内部リンク先（CDP/AI記事の終盤で「費用感はこちら」）
- ホワイトペーパー化してリード獲得導線（クラスターC素材を1本のPDFに）

**施策**:
- 新規記事の量産は **停止**（v1のP2施策は撤回）
- 既存 `estimate-complete-guide` `system-development-cost-market` `system-development-budget-control` は **品質維持リライト**（最新事例の追加、構造化データ）
- ホワイトペーパー（PDF）化を検討: 「システム開発見積もり比較チェックリスト」を `/materials` に設置 → リード獲得CTAとして機能
- CDP/AI記事のFAQ末尾から **内部リンクで誘導**

**書かない**: 規模別費用、人月単価論、見積もり比較記事（v1案）— SEO 0imp 確実なものは新規生産しない。
