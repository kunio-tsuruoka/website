6. CV計測の問題と修復（2026-05-01）

当初発見した問題
  • 全チャネルで keyEvents = 0
  • eventName 一覧に generate_lead と contact_complete が完全欠落（form_submit 6件のみ）
  • form完了率 約3%（業界水準20-30%の1桁低い）
修復内容
  1. contact-form.tsx: gtag('event',...) 直後の window.location.href = '/thanks' で navigation race が起きてビーコンがdrop していた。event_callback でflush完了を待ってから navigate するように変更（1.5秒fallback付き）
  2. thanks.astro: gtag.js 未ロード時の取りこぼし対策で window.load 再試行を追加
  3. GA4キーイベントを Admin API で確認・追加（form_submit を新規作成。generate_lead/contact_complete は既登録だった）
  4. ローカル検証 (verify-contact-cv.mjs) で 3イベント全てがdataLayerに積まれて発火することを確認済み
コミット: 8de0b65 fix(analytics): GA4 CV計測の navigation race を修正、3イベント発火を保証

7. 示唆と次のアクション（優先度順）

P2ツールの集客連結

P3分析基盤・運用

  1. LLM経由トラフィックの監視: openai/chatgpt.com の流入が始まった。dataforseo の ai_opt_* ツールで継続観測
  2. Bing最適化: 母数小だが3→9→14と急成長。Bing Webmaster Tools の整備
  3. avoid-unused-system の改善: 流入はあるが engagement 47%・平均37秒。流入クエリと本文冒頭のズレを疑う

8. 結論

「コンテンツマーケが効いているか」への端的な答え: SEO資産としては効き始めている（Organic +34%、特に3月以降）。CV貢献の定量評価は計測修復が完了した今日以降、次の3ヶ月で正確に判断可能になる。今期は「種まきが芽吹いた」フェーズ。

方向性は正しい（直近コミットでSEO/CV回りに継続的に手が入っている）。次スプリントは P0 の計測検証 → P1 の AI開発ページSEO で短期ROIを取りに行くのが妥当。
