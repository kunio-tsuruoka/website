# ピラー記事生成プロンプト

## 概要

MicroCMSに投入するピラー記事（完全ガイド）を生成するためのプロンプト集。
各カテゴリのクラスター記事を束ねる「柱」となる記事を作成する。

---

## 共通ルール

- **文字数**: 3000〜4000字
- **読者**: システム開発の発注側（非エンジニアの経営者・担当者）
- **トーン**: 専門用語を避け、平易で信頼感のある語り口。上から目線ではなく伴走者の立場
- **構成**: 見出し(h2)ごとに論点を整理し、各見出しの最後に該当クラスター記事への内部リンクを自然に挿入
- **内部リンク形式**: `[記事タイトル](/column/slug)` のMarkdownリンク
- **CTA**: 記事末尾に「まずは無料相談」への導線を含める
- **MicroCMS投入先**: `columns` エンドポイント。カテゴリは既存のものを選択

---

## ピラー記事① 見積もり（優先度：最高）

### MicroCMS設定
- **カテゴリ**: `estimate-concerns`（見積もりの不安と対策）
- **slug案**: `estimate-complete-guide`

### プロンプト

```
あなたはシステム開発の発注支援の専門家です。
以下の条件でピラー記事（完全ガイド）を執筆してください。

【記事タイトル】
システム開発の見積もり完全ガイド｜費用相場・よくある不安・失敗しない発注のコツ

【読者】
- システム開発を外注しようとしている非エンジニアの経営者・発注担当者
- 「見積もりの見方がわからない」「相場感がつかめない」「追加費用が怖い」と感じている人

【記事の目的】
- 見積もりに関するすべての論点を1本で俯瞰できる「完全ガイド」にする
- 各論点の詳細は下記のクラスター記事に譲り、ピラー記事では要点と導線を担う

【クラスター記事（必ず内部リンクを挿入すること）】
1. 「システム開発の見積もりが当てにならない理由と追加費用を防ぐ方法」→ /column/system-estimate-uncertainty
2. 「システム開発費用の相場とは？安すぎる見積もりと適正価格の基準」→ /column/system-development-cost-market
3. 「システム開発の予算オーバーを防ぐ要件コントロールと進捗管理」→ /column/system-development-budget-control
4. 「システム開発の見積もり妥当性を判断し投資対効果を高める方法」→ /column/system-estimate-validity
5. 「システム開発の初期費用リスクを抑える「ゼロスタート」モデルとは」→ /column/system-development-zero-start

【狙うキーワード】
- メイン: 「システム開発 見積もり」
- サブ: 「システム開発 費用 相場」「システム開発 発注 注意点」

【構成指示】
以下のh2見出し構成で執筆してください。各h2は300〜500字程度。

h2: システム開発の見積もりが難しい理由
  → なぜ見積もりは「あてにならない」と言われるのか。変動要因を解説
  → リンク先: /column/system-estimate-uncertainty

h2: システム開発の費用相場を知る
  → 規模別の費用感、安すぎる見積もりの危険性
  → リンク先: /column/system-development-cost-market

h2: 見積もりの妥当性を判断する方法
  → 複数見積もり比較、内訳チェックのポイント
  → リンク先: /column/system-estimate-validity

h2: 予算オーバーを防ぐための要件管理
  → スコープの決め方、変更管理の基本
  → リンク先: /column/system-development-budget-control

h2: 初期費用リスクを抑える発注モデル
  → ゼロスタート型の紹介、プロトタイプ検証の価値
  → リンク先: /column/system-development-zero-start

h2: まとめ：見積もりで失敗しないために
  → 記事全体の要約＋Beekleへの相談CTA

【文体ルール】
- 「です・ます」調
- 専門用語には必ず平易な補足を添える
- 箇条書きを積極的に活用して読みやすく
- 各h2セクションの最後に「▶ 詳しくはこちら：[記事タイトル](/column/slug)」の形で内部リンクを挿入
- 文字数: 3000〜4000字
```

## 使い方

1. 上記プロンプトをClaude/ChatGPTに投入して記事本文を生成
2. 生成されたMarkdownをMicroCMSの `columns` エンドポイントに新規記事として投入
3. カテゴリは既存のものを選択（slug案はMicroCMS側で設定）
4. 投入後 `node scripts/generate-descriptions.mjs` でmeta descriptionを自動生成
5. 各クラスター記事の末尾にピラー記事への「戻るリンク」を追加（例: 「▶ 見積もりの全体像を知る：[完全ガイドに戻る](/column/estimate-complete-guide)」）

## クラスター記事への逆リンク追加

ピラー記事を投入したら、各クラスター記事にもピラー記事への戻りリンクを追記する。
MicroCMSの各記事の本文末尾に以下を追加：

```markdown
---

📖 **このテーマの全体像を知る**
[見積もり完全ガイド｜費用相場・よくある不安・失敗しない発注のコツ](/column/estimate-complete-guide)
```
