# ADR 0002: コンテンツの目標を「本数」から「転換（CVR）」へ移す

- Status: Accepted
- Date: 2026-07-01
- Deciders: kunio (Beekle)
- Relation: ADR 0001（コラム本数の目安を「100本」に置く）の Decision-3「100本到達時点で評価し、次ADRの起点とする」を受けた後継ADR。

## Context

- 2026-07-01 時点で公開済みコラムは **101 本**。ADR 0001 が置いた「公開100本」の最低ロットに到達した（±10本の評価タイミング内）。
- **集客はもう律速ではない**。GA4セッションは 2月209 → 6月で587（倍ペース）。「効くまで書ききる」フェーズの目的は達した。
- 一方で **流入と転換の経路がズレている**。GSCクリックの約44〜53%は Gherkin/EARS 系（A-zone＝純エンジニア向け）で、ここからの実問い合わせはほぼゼロ。実際の問い合わせは header/直接や買い手意図ページから来ている。
- 2026-06 以降、**実問い合わせが来始めた**（下請け/協業含む）。律速は「流入 → 発注者リードへの転換（CVR）」に移った。
- 関連: `.claude/rules/content-strategy-goals.md`（CVR改善フェーズ・3セグメント）、`project_content_targeting_pivot`（A-zoneはCV0）、`.claude/rules/cro-strategy.md`（ツール公開・資料ゲート）。

## Decision

**コンテンツの主目標を「公開本数」から「転換（＝質の高い実問い合わせ件数）」へ移す。本数ノルマは撤廃する。**

1. **北極星指標を実問い合わせ件数に置く**（Slack到着ベース、スパム除外）。GA4 の `form_submit` はスパムを含むため主KPIに使わない。先行指標は高インプレ記事のCTR・`cta_click`・資料DL(`download_request`)・B-zone LP到達。
2. **新規記事は買い手意図（B-zone）に絞る**。費用・RFP・CDP比較・要件定義、および下請け/協業クエリ（「AI開発 外注」「RAG 開発会社」等）。**A-zone（Gherkin/EARS 等の純エンジニア向け）の新規量産はしない**。
3. **新規より既存「勝ち筋」ページの深掘りを優先**。AI Citations / GSC で引用・流入の多い買い手意図ページ（`system-development-cost-breakdown` / `how-to-write-rfp` / `cdp-product-comparison` / `requirements-definition-*` / `ai-development-cost-guide` 等）に、CTA・RTB・事例・社内ナレッジ実例を厚くする。新規1本より既存改稿のROIが高い局面と位置づける。
4. **技術記事は捨てない。ただし出すCTAを変える**。AI開発/RAG等の技術深掘りは下請け/協業（セグメントB）に効くため、これらのページには「開発パートナー/開発リソースをお探しの開発会社・SIer様」向けの相談CTAを置く（発注準備ツール誘導はBには無効）。
5. **ペースは中庸**。月数本の買い手意図記事を鮮度維持で出す。「あと何本」での進捗管理はしない。
6. ADR 0001 は本Decisionにより「本数目標は達成・クローズ」とする。量産フェーズの執行ルール（リライト/切り出しの並行）は引き続き有効。

## Consequences

### Positive

- 投資が「読まれるが売れない記事の量産」から「売れる導線の強化」に向く。
- 効果測定が本数でなく実問い合わせ＝事業成果に直結する指標になる。
- セグメント（発注者A／下請けB／学習者C）ごとにCTAを出し分けることで、技術流入も収益チャネル（B）に繋がる。

### Negative / リスク

- 実問い合わせは件数が小さく、月次のブレが大きい。短期の増減で方針を揺らさない（四半期で見る）。
- 当日の問い合わせは GA4 の処理遅延・user_id 非保持で個別ジャーニーを名指しできない。流入の裏取りは Slack 通知の「流入」ブロック＋翌日以降の GA4/Clarity で行う。
- 新規本数を絞ることで、長期のロングテール網羅は緩やかになる。B-zone のカバレッジ拡張は中期で別途進める。

## Follow-up

- ADR 0001 の Status に本ADRへの参照を追記。
- 勝ち筋ページの深掘り対象リストと、技術記事へのB向けCTA設置を別タスクで起票。
