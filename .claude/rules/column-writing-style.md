# Column writing style (AIっぽさ排除)

MicroCMS の `columns` に入稿する記事本文・ドラフトを書く時のスタイル規約。

## AIっぽい言い回しを使わない

### 冗長な主語明示
日本語では主語を落とすのが自然。「自分が」「自身が」を機械的に付けて名詞を修飾するのは生成AI特有のクセが出やすい。

- ❌ 「**自分が思う**理想のスケジュール」
- ⭕ 「希望的観測のスケジュール」「願望ベースのスケジュール」「絵に描いた餅のスケジュール」
- ❌ 「**自分が考える**ベストプラクティス」
- ⭕ 「ベストプラクティス」「考えうるベストプラクティス」

検出: `grep -E '自[分身]が(思う|考える|感じる|抱く|描く)' draft.html`

### 過剰な強調定型句
論旨と関係なく強調だけしている定型句は AI 文の典型。

- ❌ 「〜に他なりません」
- ❌ 「〜と言っても過言ではありません」
- ❌ 「まさに〜と言える存在」「いわば〜のようなもの」(連発)
- ❌ 「実は〜なのです」(段落頭で連発)

具体名詞・動詞で言い切る方が信頼できる。比喩を入れる時は1記事1〜2回まで。

### その他のAI tells
- 同段落内で同じ語尾（〜です/〜ます）が4文以上続く → 体言止めや「〜ません」「〜でしょう」を混ぜる
- 箇条書きの各項目が「〜することが大事」「〜することが重要」で終わる → 動詞で言い切る
- 「結論」「ポイント」「まとめ」を1記事に3回以上書かない

## MicroCMS は HTML 入稿（Markdown 残骸を残さない）

`columns` は HTML フィールド。**`<pre>` 以外で `**bold**` `# heading` `[text](url)` `* list` などの Markdown 記法を残さない**。サニタイザは強調記号を解釈しないので、本文にそのまま `**` 記号が表示される事故になる（実例: `things-not-to-do-pm` で `**「自分が思う理想のスケジュール」**` が本文に出ていた、2026-05-03 修正）。

代わりに:
- 強調 → `<strong>...</strong>`
- 見出し → `<h2 id="...">...</h2>` `<h3 id="...">...</h3>`
- リンク → `<a href="/path">label</a>`
- 箇条書き → `<ul><li>...</li></ul>`

ドラフト末尾で必ず確認:
```bash
grep -nE '\*\*[^*]+\*\*|^#{1,4} |^\* |^- ' drafts/<slug>.html
# ヒットゼロが正常
```

## 入稿/PATCH 前のチェックリスト（コピペ用）

```bash
# AIっぽい冗長主語
grep -nE '自[分身]が(思う|考える|感じる|抱く|描く)' file
# 過剰な強調定型句
grep -nE 'に他なりません|と言っても過言ではありません' file
# Markdown 残骸
grep -nE '\*\*[^*]+\*\*|^#{1,4} |^\* |^- ' file
# どれもヒットゼロが目安
```

ヒットしたら言い換えてから `publish-drafts.mjs` または直接 PATCH する。

## 既存記事のスキャン

全記事を再スキャンするスニペット:
```js
// scripts/scan-column-style.mjs (作成するなら)
const arr = await fetchAllColumns();
const patterns = [
  /自[分身]が(思う|考える|感じる)/g,
  /\*\*[^*]{1,40}\*\*/g,
  /に他なりません/g,
  /と言っても過言ではありません/g,
];
for (const a of arr) for (const p of patterns) {
  const m = (a.content||'').match(p);
  if (m) console.log(a.id, p, m.slice(0,3));
}
```
