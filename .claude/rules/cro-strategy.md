# CRO方針: ツールは公開、サービス資料はゲート

Beekleサイトのコンテンツゲーティング方針（2026-05-03 確定、Hide坂本・もるとから提案）:

**完全公開（メアド取得しない）**:
- 3ツール (flow-mapper / scope-manager / story-builder) およびその出力(RFPドラフト等)
- 業界別テンプレート(経理締め・営業案件・採用フローなど)
- コラム記事
- チェックリスト

**メアド取得（ゲート）**:
- 会社紹介資料(サービスデック)
- 料金/見積もり目安資料
- 事例集PDF
- サービス比較ガイド

**Why**:
- ツール/テンプレを公開ゲートで縛ると流入が死ぬ。検索→体験→ファン化→指名 のフローが切れる
- サービス資料は「Beekleを比較検討候補に入れている」段階の人が落とす → ここでメアド取れれば商談直結
- ツールで母集団を絞らず広げ、比較検討段階の濃い相手だけメアド取る、の二段階

**How to apply**:
- 新規ツール・テンプレ・チェックリスト系コンテンツ → 公開、計測のみ(GA4キーイベント化)
- 「Beekleの何を提供しているか」を見せる類の資料 → /downloads/* 専用LPでメアドフォーム経由
- コラム末CTAは記事カテゴリで分岐: ノウハウ系→ツール誘導、サービス比較系→資料DL誘導
