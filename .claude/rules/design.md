# LPデザインの色数はprimary+ニュートラルに絞る（ユーザー確定 2026-07-07）

- コンポーネントレベルの多色ローテーション（アイコン/バー/チップを紫→シアン→黄で回す）は**禁止**。2026-07-07に導入したがユーザーに「色使いが完全におかしい、色無駄に使いすぎ」と却下され、PR #86 で primary+neutral に統一した。
- 原則: **1画面1アクセント**。カード・アイコン・バッジ類は primary（Beekleパープル）系のみ。背景のリズムは neutral-50 / white の交互で作る（色付きグラデ背景は使わない）。
- secondary（シアン）/ highlight（黄）は Section コンポーネントの低opacity背景装飾（decoration-bar/dot）と、明確に意味のある注意喚起だけに残す。ボタンやラベルの塗りに使わない。
- ダークCTAセクションは bg-navy-950 + 白ボタン(主) + primary-500ボタン(副) + 白アウトライン(第三) が標準（services/[id].astro 最下部が見本）。
- design.md の「Pop Style」は背景装飾の話であり、UI要素の多色化を意味しない、と解釈する。

# LPの情報設計原則（ユーザー確定 2026-07-07）

- **一本道の原則**: LPは上から下に読むだけで訪問者の疑問が順に解ける構成にする。読む順番を選ばせるメタナビ（「まず確認/次に理解/最後に判断」型の3択カード等）は置かない。2026-07-07にservice-scan-summaryをこの理由で撤去した。
- **機械的ラベル禁止**: 「GUIDE 01」のような通し番号ラベルは付けない。見出し自体に語らせる。
- **見出しはストーリーアーク**: h1/h2だけ通読して「約束→できること→つまずき→防ぎ方→証拠→頼める範囲→作り方→試す→質問→相談」の流れが通ること。「よくある課題」「サービス内容」のような無機質ラベルでなく自然な文にする。
- **中間CTAは1箇所**（事例=信頼形成の直後）。同一バナーの繰り返しはバナーブラインドネスを招く。最下部CTAはボタン2つまで（主=相談、副=資料DL）。
- **ヒーローは左4要素**（バッジ/見出し/リード/CTA）+右ビジュアル1点。チップ・メトリクス・箇条書きパネルの併置は情報過多（ワーキングメモリ4±1）。ライト基調（白+primaryぼかし）が現行の正。
